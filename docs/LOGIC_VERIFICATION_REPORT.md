## 二、页面跳转逻辑校验

### 2.1 排行榜 → 评论详情页

| 校验点 | 结果 | 问题/修复 |
|--------|------|-----------|
| 点击菜品卡片跳转 | ⚠️ 需优化 | `goToDetail` 使用 `router.push({ name, params, state })`，路径正确；评分区域已用 `@click.stop` 阻止冒泡 |
| 返回按钮行为 | ⚠️ 需修复 | `$router.back()` 在直接打开 `/dish/101` 时可能退回站外，应改为 `router.push('/DishRank')` 兜底 |
| 刷新/直链打开 | ✅ 通过 | `history.state` 丢失时使用 `route.params.id` 生成占位菜品，避免白屏 |

### 2.2 菜品生成页提交后跳转

| 校验点 | 结果 | 问题/修复 |
|--------|------|-----------|
| 提交防重复 | ✅ 通过 | `isSubmitting` 禁用按钮，`finally` 中重置 |
| 评论接口失败兜底 | ⚠️ 需修复 | 评论失败时仍提示「发布成功」，应区分提示 |
| 跳转目标 | ✅ 通过 | 成功时 `router.push('/DishRank')`，路径正确 |

### 2.3 接口报错兜底

| 校验点 | 结果 | 说明 |
|--------|------|------|
| request 拦截器 | ✅ 通过 | 统一提取 `err.response?.data?.message`，reject Error |
| DishDetail 评论提交 | ✅ 通过 | catch 中 `alert(e?.message)`，不卡死 |
| CreateDish 评论提交 | ⚠️ 需修复 | catch 仅 console.warn，用户无感知 |

## 三、20 字限制校验

| 校验点 | 结果 | 问题/修复 |
|--------|------|-----------|
| 前端 maxlength | ✅ 通过 | CreateDish、DishDetail 均设置 `maxlength="20"` |
| 粘贴超长处理 | ⚠️ 需修复 | 部分浏览器粘贴可突破 maxlength，需在 `@input` 中截断 |
| 后端校验 | ✅ 通过 | express-validator `isLength({ max: 20 })` |
| 超长提示 | ✅ 通过 | DishDetail 有 `isOverLimit`、CreateDish 有 `formData.comment.length > 20`（maxlength 下通常不触发，截断后更稳妥） |

## 五、修复项汇总

1. **DishDetail 返回按钮**：`$router.back()` → `router.push('/DishRank')`，保证始终回到榜单
2. **CreateDish 评论输入**：增加 `@input` + `onCommentInput` 截断至 20 字，防止粘贴超长
3. **CreateDish 评论失败**：评论接口失败时 `alert("发布成功！但评论提交失败：...")`，仍跳转榜单
4. **关键逻辑注释**：为跳转、接口依赖等补充注释
